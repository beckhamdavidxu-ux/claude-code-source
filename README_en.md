## Doubao API Integration Guide

### Step 1: Register a Volcengine Account

1. Visit [Volcengine](https://www.volcengine.com/) and sign up
2. Complete identity verification (required for API access)

### Step 2: Enable Ark (方舟) Service

1. After login, go to the [Ark Console](https://console.volcengine.com/ark)
2. Click "Enable Now" if it's your first time

### Step 3: Create an API Key

1. In the Ark Console → left menu → **"API Key Management"**
2. Click **"Create API Key"**
3. Enter a name (e.g., `tencent-ai-code`) and confirm
4. **Copy and save** your API Key (format: `3d1f5117-39d7-4b84-9feb-66d8aff8b4dc`)

   > ⚠️ The API Key is shown only once — save it immediately!

### Step 4: Activate Model Services

1. Go to Ark Console → **"Model Square"** or visit directly:
   [Model Activation Page](https://console.volcengine.com/ark/region:ark+cn-beijing/openManagement)
2. Search for and activate the following models:

   | Model Name | Model ID | Recommended |
   |-----------|---------|-------------|
   | Doubao-Seed-Code Preview | `doubao-seed-code-preview-251028` | ✅ Best for coding |
   | Doubao-Seed-2.0-Code Preview | `doubao-seed-2-0-code-preview-260215` | Latest version |
   | Doubao-Seed-1.6 | `doubao-seed-1-6-251015` | General purpose fallback |

3. Accept the service agreement — the model becomes available immediately

### Step 5: Configure and Run

#### Option A: Use the Launch Script (Recommended)

Edit `claude-code-source/start-doubao.sh` and replace the API Key:

```bash
# Change line 8 to your API Key
export ANTHROPIC_AUTH_TOKEN="${DOUBAO_API_KEY:-YOUR_API_KEY_HERE}"
```

Then run:

```bash
cd ~/Desktop/claude-code1/claude-code-source
./start-doubao.sh                          # Interactive mode
./start-doubao.sh -p "Your question" < /dev/null  # Non-interactive mode
```

#### Option B: Set Environment Variables Directly

```bash
export ANTHROPIC_AUTH_TOKEN="your_doubao_api_key"
export ANTHROPIC_BASE_URL="https://ark.cn-beijing.volces.com/api/compatible"
export ANTHROPIC_MODEL="doubao-seed-code-preview-251028"
export ANTHROPIC_SMALL_FAST_MODEL="doubao-seed-code-preview-251028"
export CLAUDE_CODE_SIMPLE=1                        # REQUIRED: skip macOS Keychain
export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
export DISABLE_AUTOUPDATER=1

bun ~/Desktop/claude-code1/claude-code-source/dist/cli.js
```

#### Option C: Use cc-switch for Multi-Model Management

```bash
# Install cc-switch CLI (macOS Universal Binary)
curl -LO https://github.com/saladday/cc-switch-cli/releases/latest/download/cc-switch-cli-darwin-universal.tar.gz
tar -xzf cc-switch-cli-darwin-universal.tar.gz
chmod +x cc-switch && mv cc-switch ~/bin/
xattr -cr ~/bin/cc-switch   # Remove Gatekeeper quarantine

# Enter TUI interface
cc-switch

# Or add provider via CLI (interactive prompt)
cc-switch provider add   # Choose "DoubaoSeed" template, enter your API Key
```

## Troubleshooting

**Q: The process hangs after startup?**
A: You must set `CLAUDE_CODE_SIMPLE=1`. Without it, macOS Keychain access blocks the process.

**Q: Error: `ModelNotOpen`?**
A: The model service hasn't been activated. Go to the [Ark Console](https://console.volcengine.com/ark/region:ark+cn-beijing/openManagement) and activate the model.

**Q: Error: `Header has invalid value`?**
A: Multiple auth sources are conflicting. Use only `ANTHROPIC_AUTH_TOKEN`, not `ANTHROPIC_API_KEY`.

**Q: Non-Apple-Silicon Mac or Linux?**
A: Use the appropriate Bun binary for your platform:
- Intel Mac: `curl -LO https://github.com/oven-sh/bun/releases/latest/download/bun-darwin-x64.zip`
- Linux ARM64: `curl -LO https://github.com/oven-sh/bun/releases/latest/download/bun-linux-aarch64.zip`
- Linux x64: `curl -LO https://github.com/oven-sh/bun/releases/latest/download/bun-linux-x64.zip`

## How the Source Was Restored

The full source code was extracted from the `cli.js.map` source map in the `@anthropic-ai/claude-code` v2.1.88 npm package:

```bash
# 1. Download the npm package
npm pack @anthropic-ai/claude-code --registry https://registry.npmjs.org

# 2. Extract
tar xzf anthropic-ai-claude-code-2.1.88.tgz

# 3. Extract sources from the source map
node -e "
const fs = require('fs'), path = require('path');
const map = JSON.parse(fs.readFileSync('package/cli.js.map', 'utf8'));
const outDir = './claude-code-source';
for (let i = 0; i < map.sources.length; i++) {
  const content = map.sourcesContent[i];
  if (!content) continue;
  let relPath = map.sources[i];
  while (relPath.startsWith('../')) relPath = relPath.slice(3);
  const outPath = path.join(outDir, relPath);
  fs.mkdirSync(path.dirname(outPath), { recursive: true });
  fs.writeFileSync(outPath, content);
}"
```

The source map contains **4,756 source files** with full `sourcesContent` — a lossless restoration of all TypeScript/TSX code.
